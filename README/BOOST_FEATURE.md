# Boost機能 - サーバーブースト通知

## 概要

サーバーブーストしてくれたユーザーに感謝のメッセージを自動送信する機能です。

## 機能詳細

### 1. 本番環境での動作
- **検知方法**: Discord の `GuildMemberUpdate` イベントを監視
- **トリガー**: ユーザーがサーバーブーストを開始したとき（`premiumSince` が null → Date に変化）
- **送信先**: `features.conf` で設定されたチャンネル（本番: 1452263017348857896 = 😛雑談掲示板😉）

### 2. テスト環境での動作
- **検知方法1**: 本番と同じ `GuildMemberUpdate` イベント（実際にブーストした場合）
- **検知方法2**: テストユーザーによる疑似ブーストコマンド
  - テストユーザーID: `902878433799979078`, `1107669393049128961`
  - コマンド: `!testboost` または `!boost`
  - 任意のチャンネルで実行可能
  - コマンドメッセージは自動削除される

### 3. 送信メッセージ
```
[ユーザー名]さん、ブーストありがとう！ めっちゃ助かる！音質・高画質配信が向上して、みんなでさらに楽しめそうです！これからもコミュニティを一緒に盛り上げていこうねー！
```

## 設定

### features.conf
```conf
# Boost機能（サーバーブースト通知）
# 形式: boost=有効/無効:環境
boost=true:test   # テスト環境で有効
# boost=true:prod # 本番環境で有効
```

## テスト方法

### 1. 疑似ブーストイベントのテスト
テストユーザー（`902878433799979078` または `1107669393049128961`）で：

```
!testboost
```
または
```
!boost
```

任意のチャンネルでこのコマンドを送信すると、設定されたチャンネルに感謝メッセージが送信されます。

### 2. 本番ブーストイベントのテスト
実際にサーバーブーストを行う必要があります（テスト困難なため疑似コマンドを用意）。

### イベントフロー

#### 本番ブーストイベント
```
Discord: ユーザーがブースト
  ↓
GuildMemberUpdate イベント発火
  ↓
oldMember.premiumSince === null
newMember.premiumSince !== null
  ↓
sendBoostMessage()
  ↓
指定チャンネルに感謝メッセージ送信
```

#### テストブーストイベント
```
テストユーザー: !testboost
  ↓
MessageCreate イベント発火
  ↓
テストユーザーIDチェック
  ↓
コマンドメッセージ削除
  ↓
sendBoostMessage()
  ↓
指定チャンネルに感謝メッセージ送信
```

## 堅牢性の担保

### 1. エラーハンドリング
- すべてのイベントハンドラに try-catch を実装
- エラーログを詳細に記録（logger.error）
- 一部の失敗（メッセージ削除失敗など）は警告レベルで処理

### 2. 多重チェック
- ボットメッセージを無視
- DM を無視
- テストユーザーIDの厳密な照合
- チャンネルの存在と種類の検証

### 3. ロギング
すべての重要なイベントをログに記録：
- `boost.feature.setup`: 機能起動時
- `boost.real_event.detected`: 本番ブーストイベント検知
- `boost.test_event.triggered`: テストブーストコマンド実行
- `boost.send_message.success`: メッセージ送信成功
- `boost.send_message.error`: メッセージ送信失敗

### 4. フェイルセーフ
- チャンネル取得失敗時は静かに失敗（ログのみ）
- 設定ファイル読み込み失敗時はデフォルト値使用
- テストコマンドメッセージ削除失敗は無視（権限不足時）

## トラブルシューティング

### メッセージが送信されない
1. ログを確認: `pm2 logs yamichan-bot --lines 100`
2. チャンネルIDが正しいか確認
3. Bot権限を確認（Send Messages）
4. `features.conf` で機能が有効か確認

### テストコマンドが動かない
1. ユーザーIDが `boostTarget.js` の `testUserIds` に含まれているか確認
2. コマンドが正確に `!testboost` または `!boost` か確認（前後の空白なし）
3. ログで `boost.test_event.triggered` が記録されているか確認

### 本番ブーストが検知されない
1. `GuildMemberUpdate` イベントが発火しているかログ確認
2. Bot がギルドメンバーの更新を受け取る権限があるか確認
3. `premiumSince` の変化を正しく検知しているかログ確認
