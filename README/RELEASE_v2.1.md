# やみちゃんBot v2.1 - リリースノート

## 🎉 新機能: リプライチェーン

### 概要
Botの返信に対してリプライすることで、メンションやプレフィックスなしで自然に会話を継続できるようになりました。

### 使い方

#### Before (v2.0)
```
ユーザー: @やみちゃん こんにちは
Bot: こんにちは！やみだよ🌙

ユーザー: @やみちゃん 今日はいい天気だね  ← 毎回メンション必要
Bot: うん、いい天気だね！
```

#### After (v2.1)
```
ユーザー: @やみちゃん こんにちは
Bot: こんにちは！やみだよ🌙

ユーザー: [↑にリプライ] 今日はいい天気だね  ← リプライだけでOK！
Bot: うん、いい天気だね！

ユーザー: [↑にリプライ] 散歩でもする？
Bot: いいね！どこ行く？
```

### 機能詳細

#### ✅ 実装内容
- **リプライ検出**: Botの返信へのリプライを自動検出
- **ユーザー認証**: 元のユーザー以外のリプライは無視
- **タイムアウト**: 10分間有効
- **自動クリーンアップ**: 1分ごとに古いチェーンを削除
- **メモリ効率**: 軽量な実装（1000会話で約100KB）

#### 🔒 セキュリティ
- 他のユーザーが割り込めない
- ギルドごとに分離
- スパム検知も引き続き適用

#### 📊 パフォーマンス
- 検出速度: O(1)
- メモリ使用: 最小限
- 通常使用でパフォーマンス影響なし

### 注意事項

#### メモリについて
- リプライチェーンはメモリ上で管理
- **Bot再起動時にクリア**される
- 再起動後は新しくメンション・プレフィックスで呼び出す必要あり

#### タイムアウト
- 10分間応答がない場合、自動的にチェーンが切れる
- 切れた後は新しくメンション・プレフィックスで呼び出す

#### 複数ユーザー
- 各ユーザーごとに独立したチェーン
- ユーザーAとユーザーBが同時に会話しても混ざらない

### トラブルシューティング

#### Q: リプライしても反応しない

**確認項目:**
1. 10分以上経過していないか？
2. 元のユーザー本人がリプライしているか？
3. Bot再起動していないか？

**対処:**
新しくメンション・プレフィックスで呼び出す

#### Q: 別のユーザーと話させたい

A: リプライチェーンは個人専用です。別のユーザーは新しくメンションして開始してください。

### 既知の制限

- Bot再起動でチェーンがリセット
- 10分タイムアウト後は自動延長なし
- DM（ダイレクトメッセージ）では動作しない
