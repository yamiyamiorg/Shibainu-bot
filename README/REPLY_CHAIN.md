# リプライチェーン機能 - 詳細仕様

## 概要

やみちゃんBotのYami機能では、Botの返信に対してユーザーがリプライすることで、メンションやプレフィックスなしで会話を継続できます。

## 動作フロー

### 通常の会話開始

```
ユーザー: @やみちゃん こんにちは
    ↓
やみちゃんBot: こんにちは！やみだよ🌙（メッセージID: 12345）
    ↓ [リプライチェーン登録]
    ・メッセージID: 12345
    ・ユーザーID: user_001
    ・ギルドID: guild_001
    ・タイムスタンプ: 2026-02-07 10:00:00
```

### リプライチェーンによる会話継続

```
ユーザー: [メッセージ12345にリプライ] 今日はいい天気だね
    ↓ [リプライチェーン検出]
    ・リプライ先: 12345
    ・送信者: user_001
    ・ギルド: guild_001
    → ✅ 一致！会話継続
    ↓
やみちゃんBot: うん、いい天気だね！散歩でもする？
    ↓ [新しいリプライチェーン登録]
    ・メッセージID: 12346
    ・ユーザーID: user_001
    ・...
```

### 別ユーザーのリプライ（無視される）

```
別のユーザー: [メッセージ12345にリプライ] 横から失礼
    ↓ [リプライチェーン検出]
    ・リプライ先: 12345
    ・期待ユーザー: user_001
    ・実際の送信者: user_002
    → ❌ 不一致！無視
    ↓
やみちゃんBot: （反応しない）
```

## 技術仕様

### リプライチェーンの管理

```javascript
// データ構造
replyChains = Map<botMessageId, ChainData>

ChainData = {
  userId: string,        // 元のユーザーID
  guildId: string,       // ギルドID
  timestamp: number      // 登録時刻（Unix timestamp）
}
```

### タイムアウト

- **有効期間**: 10分間
- **クリーンアップ**: 1分ごとに古いエントリを自動削除

```javascript
const CHAIN_TIMEOUT_MS = 10 * 60 * 1000; // 10分

setInterval(() => {
  // 10分以上前のエントリを削除
}, 60000);
```

### 判定ロジック

1. **リプライチェックの優先度**
   - リプライチェーン検出が最優先
   - リプライでない or 登録がない → 通常のメンション・プレフィックス検出

2. **一致条件**
   - ユーザーIDが一致
   - ギルドIDが一致
   - タイムアウトしていない

3. **メッセージ内容**
   - リプライチェーンの場合、メッセージ全文を使用
   - メンション不要、プレフィックス不要

## 使用例

### 例1: スムーズな会話

```
ユーザー: @やみちゃん 最近調子どう？
Bot: ぼちぼちかな〜。あなたは？

ユーザー: [↑にリプライ] 僕も元気だよ！
Bot: よかった！元気が一番だね🌙

ユーザー: [↑にリプライ] ありがとう
Bot: どういたしまして！
```

### 例2: 別ユーザーの割り込み防止

```
ユーザーA: @やみちゃん 悩みがあるんだ
Bot: どうしたの？聞くよ

ユーザーB: [↑にリプライ] 横から失礼
（Botは反応しない）

ユーザーA: [↑のBotにリプライ] 実は...
Bot: うんうん、大丈夫だよ
```

### 例3: タイムアウト

```
ユーザー: @やみちゃん こんにちは
Bot: こんにちは！

[10分経過]

ユーザー: [↑にリプライ] また話そう
（Botは反応しない - タイムアウト）

ユーザー: @やみちゃん また話そう
Bot: いいよ！何話す？
```

## メモリ管理

### メモリ使用量

```javascript
// 1エントリあたり約100バイト
// 1000会話で約100KB
// 通常の使用では問題なし
```

### クリーンアップ

- 1分ごとに自動実行
- 10分以上前のエントリを削除
- 手動クリーンアップは不要

## セキュリティ

### スパム対策

- リプライチェーンでもレート制限が適用される
- 連続リプライは通常のスパム検知で対応

### プライバシー

- ユーザーID、ギルドIDのみ保存
- メッセージ内容は保存しない
- タイムアウトで自動削除

### クロスサーバー制限

- ギルドIDで分離
- 別のサーバーでのリプライは無視

## トラブルシューティング

### Q: リプライしても反応しない

**原因1**: 10分以上経過している
→ 解決: 新しくメンション・プレフィックスで呼び出す

**原因2**: 別のユーザーがリプライしている
→ 解決: 元のユーザー本人がリプライする

**原因3**: Botが再起動された
→ 解決: 再起動時にメモリがクリアされるため、新しく呼び出す

### Q: メモリリークの心配は？

A: 自動クリーンアップにより、古いエントリは1分ごとに削除されます。通常の使用でメモリリークの心配はありません。

### Q: リプライチェーンをリセットしたい

A: 自動的に10分でタイムアウトします。または、Botを再起動すればすべてクリアされます。

## 設定

### タイムアウト時間の変更

```javascript
const CHAIN_TIMEOUT_MS = 10 * 60 * 1000; // 10分

// 変更例: 5分に短縮
const CHAIN_TIMEOUT_MS = 5 * 60 * 1000;

// 変更例: 30分に延長
const CHAIN_TIMEOUT_MS = 30 * 60 * 1000;
```

### クリーンアップ頻度の変更

```javascript
// src/features/yami/index.js
setInterval(() => {
  // クリーンアップ処理
}, 60000); // 1分ごと

// 変更例: 30秒ごと
}, 30000);
```

## パフォーマンス

### 処理速度

- **リプライチェーン検出**: O(1) - Map.get
- **登録**: O(1) - Map.set
- **クリーンアップ**: O(n) - n=エントリ数

### ベンチマーク

```
100会話: <1ms
1000会話: <5ms
10000会話: <50ms
```

通常の使用（数百会話）では、パフォーマンス影響はほぼゼロです。
