# やみちゃんBot Ver3〜4 新機能紹介

このドキュメントは、Ver3・Ver4で追加・変更された機能を、メンバーへの説明やお知らせ文の作成に使えるよう、読みやすい文章形式でまとめたものです。

---

## 🆕 追加された機能

### 1. サーバー統計の大幅強化（ServerStats）

**概要**
サーバーの今の状態を表示する統計チャンネルが、Ver3以降で大幅に進化しました。単なる人数表示から、サーバーの「空気感」を伝えられるダッシュボードに生まれ変わっています。

**天気予報・植物インジケーター**
VCに参加している人数やオンライン人数をもとに、サーバー全体の活気を「☀️ 快晴・賑やか」「🌤️ 晴れ・のんびり」「☁️ 曇り・静か」「🌙 夜・ひっそり」の4段階で天気に例えて表示します。また、活気の度合いに応じて植物も「🌱 静かに成長中」から「🌸 満開」まで成長します。数字だけではわかりにくいサーバーの雰囲気を、直感的に伝えることを意識しました。

**前日比の変化率表示**
オンライン人数やVC参加人数の横に、24時間前との差分が `+12` や `-3` のように表示されるようになりました。今日が昨日より賑やかかどうかがひと目でわかります。

**VC部屋の入りやすさ表示**
現在人が入っているVC部屋それぞれに、入りやすさを示すバッジを付けて表示します。🟢 は話しかけやすい雰囲気、🔵 は全員聴き専でまったりしている、🟡 は盛り上がり中、🔴 は満員です。「どの部屋なら入りやすいかな」と迷うことが少し減ることを期待しています。

**聴き専への光（A-3）**
マイクをOFFにして静かに参加している人も、「🎧 聴き専 ← 静かに参加中」と肯定的に表示されるようになりました。声を出さなくても、一緒にいるだけでカウントされるサーバーであることを可視化しています。

**マイルストーンカウントダウン**
次の目標メンバー数（1,500人・2,000人・3,000人など）まであと何人かを常に表示します。一緒に目指している感覚を持てるようにしたいと考えました。

**サーバー開設日数・新規参加者数**
サーバーが開設してから何日目かと、直近24時間で何人が新しく参加したかを表示します。また、最後に誰かが参加したのがどのくらい前かも表示されます。

**週次ピークタイムレポート（B-2）**
毎週月曜日の深夜0時に、先週1週間でVCが最も賑わった時間帯のレポートが統計チャンネルに自動投稿されます。「このサーバーは土曜の夜が賑やかい」のような傾向がわかるようになります。データが蓄積されるほど精度が上がります。

**キーワードトレンド（A-6）**
設定したチャンネルのメッセージを1時間ごとに解析し、その時間に複数回登場した話題のキーワードをEmbedに表示します。形態素解析のようなライブラリは使わず、カタカナ語・漢字語・英語をパターンマッチで抽出する軽量な実装です。「今みんながアニメの話をしている」「GCPの話が多い」といったことが数字ではなく言葉で伝わります。

### 2. 秘密の日記へのリアクション機能（DiaryReaction）

**概要**
秘密の日記のフォーラムに新しい投稿があると、BotがGeminiを使って投稿の雰囲気や感情を読み取り、内容に合った絵文字を3つリアクションとして付けます。

**なぜ絵文字だけなのか**
コメントを返すと「Botに読まれた感」が出てしまうため、あえて絵文字だけにしています。誰かが受け取ってくれたという感覚は伝えつつ、日記のプライベートな空気を壊さないことを優先しました。

**遅延リアクション**
投稿の直後ではなく、30秒〜2分のランダムな遅延後にリアクションします。即座に反応するとBot感が強くなるため、自然に読んで反応している雰囲気を出しています。

**ぬくもり機能との共存**
あぐりーさんが開発中のぬくもり機能が使う絵文字（❤️ 💚 🫶 🤝 🌱 🪽）とは被らないよう、Geminiへの指示に除外リストを含めています。

**Ver4.0での状態**
機能の実装は完了していますが、Ver4.0ではリリースを見送り、意図的にOFFにしています。`features.conf` の `diaryreaction=false` を `diaryreaction=true:prod` に変更するだけで有効になります。

### 3. 複数サーバー同時更新の仕組み

**概要**
Ver4から、`features.conf` の設定値によって本番サーバーとテストサーバーを同時に動作させる仕組みを導入しました。

**動作ルール**

| features.conf の値 | 動作するサーバー |
|--------------------|----------------|
| `false` | どのサーバーにも反映しない |
| `true:test` | テストサーバーのみ |
| `true:prod` | 本番サーバー ＋ テストサーバー（同時） |

これにより、本番で動かしながら同時にテストサーバーでも確認できるようになります。ServreStatsは本番と同じ設定でテストサーバーの統計チャンネルにも常に表示されます。

## 🔧 変更・改善されたこと

### Nukumori機能の分離
ぬくもり機能はあぐりーさんへ移管し、このBotのコードベースから削除しました。コードとDBの設定はバックアップとして保存済みです。

### 設定IDのハードコード化
サーバーIDやチャンネルIDを `.env` に書かなくてもデフォルトで正しいIDが設定されるようにしました。環境変数での上書きも引き続き可能です。

### 時系列データベースの追加
サーバー統計を1時間ごとにSQLiteに保存するようになりました。過去のデータを使って「24時間前との差分」や「週次レポート」を生成しています。保存期間はスナップショットが30日、キーワードログが7日、参加ログが30日です。
